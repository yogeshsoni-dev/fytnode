# Future Roadmap

## Near-Term (0-3 months)
- Finalize React Native target architecture.
- Implement RN auth + profile + check-in parity flows.
- Stabilize backend contracts used by mobile.
- Add environment management and release channel discipline.

## Mid-Term (3-6 months)
- Phase out Android native app after parity + rollout confidence.
- Add backend observability and performance dashboards.
- Formalize API versioning and backward compatibility process.
- Introduce structured queue for asynchronous workloads.

## Long-Term (6-12 months)
- Add AI-assisted coaching features via isolated service layer.
- Move toward shared design system across web + RN.
- Introduce realtime capability if product requires live attendance/chat.
- Harden security posture (device trust controls, anomaly detection).

## Engineering Governance
- Keep docs under `docs/` updated per major architecture changes.
- Require migration-sensitive flow checks in release checklist.
- Track preserved business logic parity as an explicit QA artifact.
